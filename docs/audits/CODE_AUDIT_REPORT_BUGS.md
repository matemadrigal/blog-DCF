### 1. **src/dcf/projections.py** - L√≠nea 47: Bug en manejo de zeros
**Severidad:** üî¥ CR√çTICA
**Archivo:** `src/dcf/projections.py:47`
**L√≠nea problem√°tica:**
```python
if fcf_prev == 0:
    if fcf_curr > 0:
        growth_rates.append(5.0)
    elif fcf_curr < 0:
        growth_rates.append(-1.0)
    # else: both zero, skip (no meaningful growth)
```

**Problema:**
Cuando `fcf_prev == 0` y `fcf_curr == 0`, el c√≥digo hace `continue` sin agregar nada a `growth_rates`. Esto causa:
- **Desincronizaci√≥n de √≠ndices**: `growth_rates` tiene menos elementos que `fcf_history - 1`
- **Error en funciones downstream**: Cualquier c√≥digo que asuma `len(growth_rates) == len(fcf_history) - 1` fallar√°

**Ejemplo del bug:**
```python

# Expected: 3 growth rates [-1.0, 0.0, infinity]
# Actual: 2 growth rates [-1.0, 5.0]  # MISSING one!
# Index mismatch!
```

**Fix requerido:**
```python
if fcf_prev == 0:
    if fcf_curr > 0:
        growth_rates.append(5.0)
    elif fcf_curr < 0:
        growth_rates.append(-1.0)
    else:  # both zero
```

### 2. **src/dcf/model.py** - L√≠nea 74-75: Terminal Value con FCF negativo
**Severidad:** üî¥ CR√çTICA
**Archivo:** `src/dcf/model.py:74-75`
**L√≠nea problem√°tica:**
```python

# ‚ùå BUG: Si last_cf es negativo, TV ser√° negativo
```

**Problema:**
Si el √∫ltimo FCF proyectado es negativo (empresa en distress), el c√°lculo del Terminal Value producir√° un valor negativo, lo cual:
- **No tiene sentido financiero**: Una empresa con FCF negativo perpetuo no puede valer algo
- **Deber√≠a valer 0 o ser flag de error**: El modelo DCF no es apropiado
- **Puede dar valuaciones absurdas**: EV negativo cuando suma PV de FCF positivos + TV negativo

**Ejemplo del bug:**
```python
cf_list = [100, 80, 60, 40, -20]  # Distressed company
last_cf = -20
terminal_value = -20 * 1.025 / 0.08 = -256.25  # ‚ùå Nonsense
```

**Fix requerido:**
```python
last_cf = cf_list[-1]

# [AuditFix] Handle negative terminal FCF
if last_cf <= 0:
    warnings.warn(
        f"‚ö†Ô∏è  Terminal FCF is non-positive ({last_cf:,.0f}). "
        f"Setting Terminal Value to 0. DCF may not be appropriate for this company.",
        UserWarning
    )
    terminal_value = 0.0
else:
    terminal_value = last_cf * (1 + perpetuity_growth) / spread
```

### 5. **src/dcf/ifrs16_adjustments.py** - L√≠nea 98: Perpetuity assumption sin validaci√≥n
**Severidad:** üü° MEDIA
**Archivo:** `src/dcf/ifrs16_adjustments.py:98`

**L√≠nea problem√°tica:**
```python
# Simple capitalization: PV = Annual Payment / r
# Assumes constant lease payments in perpetuity
estimated_liability = annual_lease_expense / discount_rate  # ‚ùå Simplification extrema
```

**Problema:**
- **Leases NO son perpetuos**: T√≠picamente 3-10 a√±os
- **Formula correcta**: PV de anuidad finita = PMT √ó [1 - (1+r)^-n] / r
- **Sobreestima el liability**: Perpetuity formula da valores muy altos

**Fix requerido:**
```python
# Better: Assume finite lease term (conservative 7 years average)
average_lease_term = 7
discount_rate = 0.05

# Annuity formula: PV = PMT √ó [1 - (1+r)^-n] / r
pv_factor = (1 - (1 + discount_rate) ** -average_lease_term) / discount_rate
estimated_liability = annual_lease_expense * pv_factor

metadata["lease_term_assumed"] = average_lease_term
metadata["formula"] = "Finite annuity (not perpetuity)"
```

### 6. **src/dcf/wacc_calculator.py** - L√≠nea 596: ERP division by zero
**Severidad:** üü° MEDIA
**Archivo:** `src/dcf/wacc_calculator.py:599`

**L√≠nea problem√°tica:**
```python

    # METHOD 1: Adjust beta (Damodaran method)
    # Œ≤_adjusted = Œ≤ + Œª √ó (CRP / ERP)
```

**Problema:**
La condici√≥n `equity_risk_premium > 0` previene divisi√≥n por cero exacto, pero no protege contra:
- **ERP muy peque√±o** (0.001): Beta adjustment = 10 √ó CRP/0.001 = 10,000 √ó CRP (absurdo)
- **Valuaciones en mercados de baja volatilidad**: ERP puede ser < 2%

**Fix requerido:**
```python

if apply_country_risk_to_beta and country_risk_premium > 0 and equity_risk_premium > MIN_ERP:
    # Safe to divide
    beta_crp_adjustment = lambda_exposure * (country_risk_premium / equity_risk_premium)
else:
    # ERP too low or negative, fall back to separate addition
    beta_crp_adjustment = 0.0
```
